# base64

A drop-in fast path for standard base64 (`StdEncoding`, padded), byte-identical
to `encoding/base64`. The bulk of **encoding** runs a SIMD kernel generated by
[go-asmgen](https://github.com/go-asmgen/asmgen); the short tail + padding reuse
the standard library, so output matches exactly.

```go
s := base64.EncodeToString(data)      // same bytes as encoding/base64.StdEncoding
```

| op | amd64 | arm64 / loong64 / riscv64 | decode |
|---|---|---|---|
| encode | **SSE2/SSSE3** (Lemire) | scalar (stdlib) — NEON/LSX/RVV planned | scalar (stdlib) — SIMD planned |

The encoder is Lemire's vectorised base64: `PSHUFB` spreads 12 input bytes across
four 24-bit lanes, two multiplies pull out the 6-bit indices, and a `PSHUFB`
offset-LUT maps each to its ASCII byte (constants emitted via go-asmgen's
`emit.File.Data`). Verified against `encoding/base64` (table + fuzz).

> Note: the arm64 NEON encode wants an integer vector multiply for the 6-bit
> extraction; Go's arm64 assembler lacks `VMUL` today (a shift-based path or the
> pending upstream `VMUL` patch resolves it).
